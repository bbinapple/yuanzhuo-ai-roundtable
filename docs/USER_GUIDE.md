# 圆桌会议使用说明

圆桌会议是一个 AI 辩论工作台。你输入一个议题，多个 Agent 会从不同角度拆解问题，最后由秘书整理总结、待办和评分。

## 1. 先配置自己的 API

打开页面后，点击右上角 `设置`，找到 `API 设置`。

必填：

- `API Base URL`：你的模型接口地址，例如 `https://your-api.example.com/v1`
- `API Key`：你自己的 API Key

建议填写：

- `分析型默认模型`：用于 A 辩手，适合长线分析和风险拆解
- `执行型默认模型`：用于 B 辩手，适合落地、成本、ROI 判断
- `秘书默认模型`：用于总结、待办、标签和评分；不填则跟随分析型模型

说明：

- API Key 只保存在你当前浏览器里。
- 网页不会使用站长的 API Key。
- 正式公网使用时，请确认网址是 HTTPS。

## 2. 选择 Agent

点击右上角 `Agent管理`。

`本轮辩手` 用来设置当前这场辩论：

- `A 辩手`：默认是分析型辩手
- `B 辩手`：默认是执行型辩手
- `C 辩手`：可选第三辩手，开启后才参与辩论

`Agent 库` 用来管理你自己的长期 Agent：

- 新增 Agent
- 编辑 Agent 的模型和 prompt
- 删除不用的自定义 Agent

内置 Agent 被编辑时，会复制成一个自定义版本，不会直接覆盖系统默认角色。

## 3. 开始一场辩论

在输入框里写议题，例如：

```text
这个 ecommerce 产品值得做吗？
```

点击 `开始` 或 `开始辩论`。

如果开启 `预热`，系统会先收集背景、争议点和关键事实，再让辩手开始讨论。

## 4. 辩论中可以做什么

辩论过程中可以：

- 暂停或继续
- 插入自己的补充意见
- 上传文本、图片或 PDF 附件
- 从资料中抽取议题
- 复制整段对话
- 中途导出 Markdown 快照

系统会每隔一段轮次暂停，方便你决定继续、插话或散会总结。

## 5. 散会总结

散会后，秘书会自动生成：

- 核心共识
- 主要分歧
- 下一步建议
- 待办清单
- 标签
- 辩论评分

这些结果会保存到历史议题里。

## 6. 历史和导出

点击右上角 `历史` 可以查看之前的议题。

历史里可以：

- 继续辩论
- 查看总结和待办
- 勾选待办完成状态
- 删除不需要的记录
- 导出 Markdown

议题一开始创建就会进入历史列表，避免辩论中断导致数据丢失。

## 7. 常见问题

### 提示 API Key 未配置

去 `设置 -> API 设置` 填写自己的 API Key。

### 提示模型不可用

说明你填写的默认模型不在当前 API 的模型列表里。请确认：

- Base URL 是否正确
- API Key 是否有效
- 模型 ID 是否拼写正确

### 直接打开 HTML 文件后功能不完整

请通过服务地址访问，例如：

```text
http://127.0.0.1:8891/
```

不要直接用 `file://` 打开本地 HTML 文件。直接打开只能看到部分页面，无法完整调用后端接口。
